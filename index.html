Session

Namespace for SessionHandling

/session/get-LoginToken get

If you want to interact with the API, you first have to request a LoginToken. Within 1 Minute you have to use this LoginToken to Login to the API. If you don't login in this timespan, the LoginToken get's invalid. After a login succeeded, you get a SessionToken to fire up requests to the API.

get

get /session/get-LoginToken

  • Request
  • Response

Headers

  • APIKey: required (string - minLength: 36 - maxLength: 36)

    API Key for using the API

    Example:

    590df105-200c-4851-aa9c-d5a10ea6c8f0
  • APIUser: required (string)

    API User using the API, not the Login-User!

    Example:

    IOSDeveloper
  • APIPassword: required (string)

    API Password for using the API, not the Login- Password!

    Example:

    xa4g&z76!

HTTP status code 200

OK, here is the LoginToken to login to the API

Body

Type: application/json

Example:

{
  "LoginToken":"fb25f168-68f7-45f3-b029-3491b6c7ee8d"
}

HTTP status code 500

Error, see description

/session/set-SessionTimeout post

To interact with the API you need a valid SessionToken. By default the inactivity-time of a sesion is 60 minutes. So if a user does not interact with the API for 60 minutes the session get's invalid and the user has to Login again (and before this, he has to request a new LoginToken). You can set the session-timeout to 0 (the session expires immediately, x (any int value, the session expires after x minutes).

post

post /session/set-SessionTimeout

  • Request
  • Response

Headers

  • SessionToken: required (string - minLength: 36 - maxLength: 36)

    SessionToken

    Example:

    8dfb0f07-6f6a-4dfd-9041-adca472649d2

Query Parameters

  • newsessiontimeout: required (integer)

    The value for the new session-timeout in minutes

    Example:

    15

HTTP status code 200

OK, SessionTimeout was set

Body

Type: application/json

Example:

{
  "Message":"OK, new SessionTimeout set to 125"
}

HTTP status code 500

Error, see description

/session/set-DataMode post

--Concept!--
The API supports three DataModes. If you set the value to 1 (default) you will interact with the life-database, if you set the value to -1 you will interact with the PreProd-database and if you set the value to 0 you only get example-data.

post

post /session/set-DataMode

  • Request
  • Response

Headers

    SessionToken

    Example:

    8dfb0f07-6f6a-4dfd-9041-adca472649d2

Query Parameters

    New DataMode value
